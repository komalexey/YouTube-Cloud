# YouTube-Cloud

В репозитории теперь есть перенос логики `coder.py` на C++ с консольным интерфейсом и архитектурой, удобной для дальнейшего добавления Qt GUI.

## Что есть в проекте

- `coder.py` - исходная Python-версия.
- `include/youtube_cloud` - заголовки C++-ядра.
- `src/core` - алгоритмы кодирования и декодирования.
- `src/cli/main.cpp` - консольный вход.
- `Makefile` - рабочая сборка в текущем окружении.
- `CMakeLists.txt` - основа для будущей интеграции с Qt.
- `docs/architecture_ru.md` - заметки по архитектуре, GUI и оптимизации.

## Зависимости

Для текущей C++-версии нужны:

- `clang++` или `g++` с поддержкой C++20
- `make`
- `ffmpeg`
- `ffprobe`

`OpenCV` в текущей реализации не нужен: работа с видео выполнена через внешний `ffmpeg`, а генерация/чтение пикселей сделана внутри C++-ядра. Это упрощает переносимость и снижает связность перед будущим Qt GUI.

## Сборка

```bash
make
```

Готовый бинарник:

```bash
build/youtube_cloud
```

## Использование

Кодирование:

```bash
./build/youtube_cloud encode FILENAME.xxx FILENAME.mp4
./build/youtube_cloud -e FILENAME.xxx FILENAME.mp4
```

Декодирование:

```bash
./build/youtube_cloud decode FILENAME.mp4 OUTPUT_DIR
./build/youtube_cloud -d FILENAME.mp4 OUTPUT_DIR
```

Если второй аргумент не указан:

- для `encode` будет использован `output.mp4`
- для `decode` будет использована текущая папка

## Ключ шифрования

Можно создать в корне проекта файл `key.txt` и записать в него ключ любой длины.

- если `key.txt` найден и не пустой, включается XOR-шифрование
- если файла нет или он пустой, шифрование отключено

Поведение сделано совместимым с логикой исходного `coder.py`.

## Что сохранено из Python-версии

- разрешение кадров `1920x1080`
- `6 FPS`
- угловые маркеры
- заголовок формата `FILE:<name>:SIZE:<bytes>|`
- маркер конца данных
- `5` защитных кадров
- автоматическое восстановление имени файла при декодировании

## Дальше: GUI на Qt

Рекомендуемый следующий шаг:

1. Оставить `src/core` как независимую библиотеку.
2. Добавить отдельную цель `gui/` на Qt Widgets или Qt Quick.
3. Вызывать кодирование и декодирование из фонового потока (`QThread` или `QtConcurrent`).
4. Прогресс и логи пробрасывать в интерфейс через адаптер к `ProgressSink`.
